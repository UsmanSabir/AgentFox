# Git Skill

## Overview

Provides version control operations for managing code repositories using Git. Essential for any development workflow.

## Capabilities
- Version Control
- Change Tracking
- Branching Strategy
- Collaboration
- Repository Management

## Tools Provided
- `git_commit`: Create commits with messages
- `git_push`: Push commits to remote repository
- `git_pull`: Fetch and merge from remote
- `git_branch`: Manage branches (list, create, delete)
- `git_status`: Check repository status
- `git_log`: View commit history

## When to Use

Use this skill for any version control operations, including:
- Creating feature branches for new work
- Committing changes with meaningful messages
- Merging branches back to main
- Pushing changes to remote repositories
- Tracking commit history and changes

## Best Practices

- Always commit with meaningful messages (use conventional commits format)
- Use feature branches for new work instead of committing to main
- Pull before pushing to avoid conflicts
- Review git status before committing
- Use git log to track changes and understand history
- Keep commits focused on single logical changes
- Use branches for organizing work by feature, bug fix, or release
